右移运算 RSH

1往右边1一位是0 无负数符号位移
2右边移动1位就是1，这个没啥可说的就是二进制的移位置而已，不头晕，就是负数 也就是有符号的移位有点绕，曾经明白了，现在就又糊涂了。
-1 的二进制位全是1，右移动一位依然是-1 这符号位继承了

左移运算 LSH

1左边移动一位就是2，这也没啥可说的。
来看看-1左移动一位 移动之后补0， 为什么计算器结果得到的是-2呢？

原理如下:
强制转换是用补码运算 在 按位与也是用补码的方式，但是正数 补码原码一样
原码-反码—补码
补码做运算结果也是补码
电脑内存存的是补码

无符号左右移 负数先从原码-反码-补码-然后补码（也就是对补码进行取反）的符号改为0

有符号左右移 负数先从原码-反码-补码-然后补码的符号改为照填,因为计算机运算是补码运算
左移 左边最高位丢弃，右边移空的补0，
右移 左边移空的左边最高为是1就补1，是0就补0，移动几位补几位，结果肯定是正数
右移负数 所以左边的是补1.

########### 什么是反码
正数的反码是其本身
负数的反码是在其原码的基础上, 符号位不变，其余各个位取反.

########### 什么是补码
正数的补码就是其本身
负数的补码是在其原码的基础上, 符号位不变, 其余各位取反, 最后+1. (即在反码的基础上+1)
[+1] = [00000001]原 = [00000001]反 = [00000001]补
[-1] = [10000001]原 = [11111110]反 = [11111111]补

=================这里操作 左边移动1位，
1 1 1 1 1 1 1 0 原码

1 0 0 0 0 0 0 1反码[-1] 符号位不变，其它全部取反

1 1 1 1 1 1 1 0补码[再次取反 但是还需要+1] 这里表示-2，但是人类是看不太懂的。

-2 取反=1 的原因也知道了吧，因为 -2 1 1 1 1 1 1 1 0 如果取反之后那么前面全是0 最后面一个是 1.
