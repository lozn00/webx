前言

引入sdk

引入sdk分为2种，一种是下载sdk包或者下载eclipse demo项目， 另外一种是在线添加依赖

引入方式1-下载sdk包

首先下载sdk jar包，或者android studio开发工具引入依赖。
sdk的jcenter地址如下:，第一个地址打不开可以尝试第二个下载地址
https://jcenter.bintray.com/cn/qssq666/robot_sdk/
https://bintray.com/luozheng/maven/robotsdk

下载最新版的,打开这个地址有0.5,也有可能有1.1 2.2根据数字排号，下载最大的即可。
比如0.1的包列表地址如下:
https://jcenter.bintray.com/cn/qssq666/robot_sdk/2.4/
包含如下文件

那么进行开发肯定是下载最短的jar文件, sources表示源文件,没必要,javadoc则是一个说明书一样的东西,用api的时候就容易理解一点.
那么这里最短的是robot_sdk-2.4.jar
sources.jar是源代码，不要这么做，因为每次sdk升级你累成狗，不现实的，而且你不小心改动了然后生成插件会导致和我的机器人不兼容。 javadoc.jar是一个html文件包，实际上就是文档了，解压之后可以打开,index.html浏览说明。

引入方式2-在线SDK依赖

android studio 等支持添加依赖的可以这样写

jcenter

compileOnly 'cn.qssq666:robot_sdk:2.4'
如果是老版本gradle,则应该改成provided compileOnly 'cn.qssq666:robot_sdk:2.4
目测aide2018年11月3日 11:48:46截止 应该改成provided

下载完成之后添加到自己的工程中去就行了，使用最新sdk要配合最新的机器人使用，否则会引发崩溃，插件不会正常加载等问题。。
切记引入方式就是不要打包插件 sdk api 进入apk,dex,否则就会出现问题。所以上面的引用方式叫compileOnly

打包

打包可以生成apk包，也可以生成jar包(其实还需要通过指令转成dex),dex包，打包之后一定要放到插件的指定目录然后在插件列表刷新。或者通过插件管理器界面进行安装。
机器人版本号60以及以前是存放在自己的安装目录，可以查看简书的另外一篇文章。另外一篇文章在情迁软件标签
之后的版本我打算放到/sdcard/qssq666/robot_plugin目录下.生效方式是重启生效或者调用命令配置重载生效或者在插件列表下拉刷新生效。

总结

1、掌握如何使用最新版SDK进行开发,
2、注意sdk最新版的情况下也要让QQ机器人保持最新版
3、出现问题建议学会看logcat,不过最新版插件激活出现错误都可以通过gui界面进行查看。哪怕内部崩溃也可以使用try catch然后在插件列表选中被标记激活无效的插件查看错误详情。
4、添加引用的方式切记不要把sdk打包进入你的插件，具体方式可以下载jadx插件apk,然后看看sdk里面的api有没有打包进去，（针对eclipse的本地集成方式）在线gradle集成使用provided或者compileOnly肯定是没有问题的。
开发尽可能兼容低版本sdk插件，但是同时各位测试的时候最好使用最新版机器人进行测试，否则有些功能api无法使用，比如最近新增了发送图片的api,也新增了网络请求的api,运行在主线程的api.
