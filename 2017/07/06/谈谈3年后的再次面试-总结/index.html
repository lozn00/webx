谈谈3年后的再次面试总结

2017-07-07

Loading次访问

文章字数：2431,阅读全文大约需要9分钟

我的状态

跳槽阶段,正式离职第一天也就去面试了,但是我前期 我却一直没有去复习那些理论，或者说只是看了看，并没有在心里问自己是否可以背下来,
那么前面是因一直在上班，在研究项目，或者晚上研究视频播放的问题 ，并没有把那些基础，理论的东西摆在第一位，我认为我们真正需要的是应用开发的技术，坑理论都是空架子，而且我也打算休息一段时间，所以这次面试失败也是注定的咯。

我本来是还是抱着试试的态度投了几个简历，上个月投了一次用在线简历没效果，这次使用附件简历，果然效果不一样了，有面试机会了

态度方面

另外我的态度呢，也是比较懒散，并没有把面试当回事，下班也就继续弄电脑搞公司的或者比较高深一点安卓方面什么的,不愿意选择临时抱佛脚的方式去搞那些基本上用不到的java基础或者本来不是很懂数据结构之类的， 也想看看运气，会不会能遇到跟第一家公司一样，不会拿理论去怼人，也不用做面试题，做的项目多就ok,

就这样毫无准备的去面试高薪了，深圳科技园，还是我第一次去，我有时候会这样想，是不是这里所有路人都是程序员？？

面试情况

进入面试发现，但是情况并不是这样，能开发很多项目并不代表什么？ 能清楚描述这些东西才是牛逼。另外我给自己卖了坑，自己说的理解和精通，人家照简历上问，结果面试官也懵逼咋都答不上来，的，自己把自己给埋葬了所以 呢 今天上午那些什么原理，一问三不知.结果也是很悲惨

问了什么

来了2个技术面试我，第一个面试面试跟我谈我的项目的时候比较轻松因为的确是我做的，n多个项目，但是问到这些系统的东西我就说不出所以然了。
第一个面试的层面比较浅
handler内部的实现原理,我就简单的答了一个loop, 实际上我是知道里面有一个队列轮循，不断的取数据的。但是面试的时候我讨厌这个，我就回答了一个loop就支支吾吾说不知道了，希望各位别学我说不知道，跟一个傻逼一样，只会over..

然后又问 服务的启动方式，我又懵逼，这是靠啥子？我有懵逼了？我只会写项目啊，什么是启动方式？我就继续追问，怎么说？然后才理解面试官是想考察，我对绑定服务和启动服务的理解，之后聊的还好，说找他的组长来面试我

红黑树的什么散列来着是什么原理，这个我更加不知道，我直接说不知道。

然后又问android 内部是怎么垃圾回收的，如何根据关联确定是否被回收的。。 我又懵逼了

然后又问链表和数组的原理，我特么又懵逼， 说真的说链表是个什么东西真不清楚，因为这个东西不是我写的，我只知道他是一个东西可以往前面插后面插，很方便。
数组是分配死的空间， 我依然不知道怎么描述，又懵逼，over.
又问了冒泡排序的实现原理
我只知道是从0到最后面 这样循环交换 0 和1 1 和2 如此往复，我很久没写了，我几年前还会的，我又over了
然后又问快速排序的实现，我又懵逼了，复习了一下，好像是先从开始和最后一个往中间比较，当比较到下标一下的时候继续 折半比较，这样的的排序就改良的冒泡排序，不知道我说的对不对，我记得我几年前是轻轻松松就能写出来的，不知道哪个老师教得了，教的贼好，如今还给老师了，
之后又问了时间复杂度 和空间复杂度，这懵逼了，这个术语我真不知道，我也只能背诵了，我百度了下感觉就是一个公式，拜托我数学不好，机记性也不好，给我10次复习的机会，我照样回答不出来，我也是绝对不会吧他背熟的。
又问 view的绘制过程，又懵逼，
我只知道大概是 measure onMeasure, layout onLayouer draw ,onDraw 但是我面试回答的时候直说了measure onMeasure,说实话这种东西我还是喜欢做笔试题了，我一说话就完全不知道该说什么了，

最后可想而知，虽然我做过n多个项目，但是理论没有一个满意的， 说今天最晚下午出结果，等通知 ，我明白 我over了， 搞这种开发最怕问理论了。。。

下午的面试大公司

概述

下午是逆向的面试，不过我拿的是安卓开发的简历，问了一下eventbus的实现原理，我懵逼了，我还真说不出，因为我确实没怎么看 。 。
那么看了一下我的一些作品，公司也说他们想搞什么作品

公司的期望

经过面试，发现还是期望比较专业的公司逆向开发，但是也认同我的兴趣行为,有兴趣一切都很好说.另外说我还有一点欠缺,当然咯我也知道欠缺在哪里,那就是so方面的了,so的hook,或者调试我很少研究挑战。

经历

经过hr面试，技术面试，技术总监面试， 也是这样，就问了一下如何自动加好友什么的 ，技术都很随意很轻松吧,都是30多岁到40岁的的面试官，开玩笑也是特为开，感觉像老司机，问有没有女朋友，我们公司女生多 (确实多,公司分2册 左右都跟教室一样，这么多开发有点夸张了)，哈哈，最后技术总监很开心的说辛苦我下午跑了一趟了挺有礼貌的

面试成功打电话说可入职

但是经过电话沟通,虽然工资挺高的,和上午的一样，但是我 我对税后和税前 以及试用期3个月不满意 ，也有点虚，单纯搞java蹭逆向并不能提高多少技术，搞底层so逆向呢我又不会，如果遇不到大牛的话，我去公司基本上废了，能不能过不一定，毕竟逆向是做反对抗，遇到变态的对抗，就间接的否定了一个人的逆向能力，我拒绝了，当然他说里面也有安卓,但是我确实是面试的逆向,问的也没啥含量吧,不过我也喜欢这样子哈哈,就问能不能做之类的简单粗暴。。。。。。。

总结

面试还是需要复习理论 以及基础知识

面试还时有可能被搞得做面试题

面试还需要端正态度,哪怕不会也要想办法去说,而不是说不清楚 不懂得之类的直接逃避.
自己写的精通自己要把这个坑填上,要确保面试的时候能说出来,问到这个说不出来死的很惨

最后我想说，或许这就是真的残酷的理论面试这是我第一次经历这种高深描述的面试

口才/情商也很重要 不会理论不背也没办法，有时候记性好的人更能找到好工作，我见过牛逼的人，能把死的说成活的,打不出来的问题都可以吹得一套一套的，不懂得技术的面试官就感觉是大神的样子.

记性 好的人，直接把这些东西背输也是很ok的，对于这种喜欢问理论 描述原理的完全不怕了，我的确看过底层源码，

我很幸运第一家公司不给我来这种东西面试我，直接信任我就让我开干了，另外再比比税前税后，感觉似乎现在是行情真的不好了,或者或许第一家也是传说中的大公司 ??。
